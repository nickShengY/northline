# Event Type Catalog (v1)

This catalog extends the current implemented event list and reserves names for upcoming modules. Event payload schemas live in `packages/shared` and should be versioned via `schema_version`.

## Versioning rules

1. Event names are immutable once released.
2. Breaking payload changes require new `schema_version`.
3. Corrections are represented as new events, not in-place edits.
4. Unknown event types must be safely ignorable by old clients.
